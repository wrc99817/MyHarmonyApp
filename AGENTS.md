# AGENTS.md

## 项目简介

本项目是一个纯血鸿蒙麻将记账 App，暂名“雀账”。第一版目标是做成好友群组同步账本：麻将玩家可以创建群组、邀请牌友、创建牌局、手动录入每局四人输赢，并查看明细、结算统计和同步状态。

技术方向：HarmonyOS NEXT Stage 模型、ArkTS、ArkUI。本地优先保存数据，后续接入 HarmonyOS 关系型数据库与 AppGallery Connect 认证/云数据库，实现多端同步与权限控制。

## 当前计划

- V1：登录、建群、邀请加入、创建牌局、手动录入输赢、合计校验、统计、基础同步。
- V1.5：账单分享、常用玩家、历史筛选、离线同步冲突提示。
- V2：会员权益、云备份、多群管理、高级统计、Excel/PDF 导出。

## 当前功能

- 已搭建 HarmonyOS NEXT Stage 模型工程，使用 ArkTS、ArkUI 和 Hvigor 构建。
- 已接入 MVP 数据模型：用户、群组、成员、牌局、单局记录、同步操作日志。
- 已实现本地内存仓储和模拟同步服务，支持后续替换为 HarmonyOS 关系型数据库与 AppGallery Connect 云数据库。
- 已按 `prototype/prototype-web` 的 Web 原型重构主界面：牌局工作台、当前账面、记账输入、进出账历史、统计、牌友、我的。
- 已实现底部悬浮胶囊导航和中间“+”新建牌局入口，新建牌局使用半屏 Sheet 形态。
- 已支持单家收分记账、金额合计校验、账面统计、历史展示、模拟同步。
- 当前可在 DevEco Studio / Pura 90 HarmonyOS 模拟器中构建、安装并启动。

## 工作原则

- 保持纯血鸿蒙路线，优先使用 ArkTS、ArkUI、Stage 模型和鸿蒙官方能力。
- 功能实现先保证核心记账链路稳定，再做复杂规则和商业化能力。
- 每次功能变更后，应尽量构建并运行到本地模拟器或可视化预览，让用户能直观看到当前 App 形态。
- 安装 DevEco Studio、SDK、模拟器、命令行工具等软件前，必须先询问用户安装路径，方便归类管理。
- 不随意引入跨端框架或非鸿蒙 UI 技术；如仅为临时视觉验收，需要明确标注。
- 修改数据模型、同步策略、权限逻辑时，要同步更新说明文档或相关注释。
- 项目对话中如出现新的工作原则，应及时更新到本文件。
- 项目根目录路径不要包含空格，避免 DevEco/Hvigor 等工具链无法打开或构建。
- 项目原型图通过 `prototype/prototype-web` 以 Web 方式可视化维护；实际鸿蒙 App 界面与功能应优先按照该 Web 原型开发。
