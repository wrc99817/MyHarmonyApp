# 纯血鸿蒙麻将记账 App 方案

## Summary
第一版定位为“好友群组同步账本”：线下麻将玩家创建牌局账本，邀请牌友加入，每局手动录入输赢金额，所有成员可同步查看明细、总账和统计。后续再迭代规则模板、商业化会员和完整运营能力。

技术底座采用纯血鸿蒙：HarmonyOS NEXT Stage 模型、ArkTS、ArkUI；本地优先存储加云端同步。参考官方方向：[Stage 模型](https://developer.huawei.com/consumer/cn/arkui/arkui-stage/) 和 [AppGallery Connect 云数据库](https://developer.huawei.com/consumer/cn/agconnect/cloud-base/)。

## Key Features
- 群组账本：创建麻将群、邀请成员、成员昵称/头像、群内牌局列表。
- 牌局记录：创建一场牌局，设置玩家、底分/备注，按局录入每个玩家输赢金额。
- 自动校验：每局四人输赢合计必须为 0；支持撤回、修改、删除，并保留操作记录。
- 结算看板：按牌局、按日期、按玩家汇总总输赢、场均输赢、最大单局、最近走势。
- 同步协作：成员进入同一账本后实时或准实时看到更新；离线可录入，联网后同步。
- 基础导出：第一版支持账单截图/分享文本，后续增加 Excel、PDF、云备份。
- 隐私与风控：金额仅群成员可见；删除群、退出群、账本权限需要明确提示。

## Architecture
- 客户端：ArkTS + ArkUI，采用 Stage 模型；页面包括首页账本、牌局详情、录入局数、统计、成员管理、设置。
- 状态管理：页面内用 ArkUI 状态，跨页面共享用轻量 Store/Repository 分层，避免 UI 直接访问数据库。
- 本地数据：使用 HarmonyOS 本地关系型数据库保存群、成员、牌局、局记录、同步队列，保证离线可用。
- 云端同步：使用 AppGallery Connect 认证服务做登录，云数据库做群组账本、多端同步和权限控制。
- 数据模型核心：
  - `User`：用户基础信息。
  - `Group`：麻将群/账本。
  - `GroupMember`：成员角色与权限。
  - `GameSession`：一次牌局。
  - `RoundRecord`：单局输赢明细。
  - `LedgerOperation`：修改、删除、撤回等审计记录。
- 同步策略：本地写入后生成待同步操作；云端成功后标记已同步；冲突按“操作时间 + 操作版本号”处理，金额记录修改保留历史。

## Iteration Plan
- V1 MVP：登录、建群、邀请加入、创建牌局、手动录入输赢、汇总统计、基础同步。
- V1.5：账单分享、牌局模板、常用玩家、历史筛选、离线同步冲突提示。
- V2 商业版：会员权益、云备份、多群管理、高级统计、导出 Excel/PDF、规则模板市场。
- V3 专业版：地方玩法辅助计算、战绩排行榜、AI 复盘摘要、桌面卡片/服务卡片。

## Test Plan
- 单局金额校验：四名玩家输赢合计不为 0 时禁止保存。
- 多人同步：A 创建牌局，B/C 能看到更新；B 修改后 A 能收到变化。
- 离线场景：断网录入多局，恢复网络后正确同步且不重复。
- 权限场景：非群成员不能读取账本；普通成员不能删除群。
- 数据可靠性：修改/删除记录可追溯，统计结果与明细一致。
- 鸿蒙适配：手机、折叠屏、平板基础布局适配；深色模式和字体放大不遮挡。

## Assumptions
- 第一版采用“手动输赢记账”，不做完整番型算分。
- 第一版即支持好友群组同步，不做纯单机版本。
- 登录和云同步优先使用华为生态能力，保持纯血鸿蒙技术路线。
- 商业化能力暂不进入 MVP，只预留账号、会员状态和云端配置扩展点。
